# Risk Register

This document identifies potential risks associated with the Comprehensive GST-Compliant Accounting Suite and outlines mitigation strategies.

| Risk ID | Category       | Risk Description                                                                                                                                                     | Impact | Likelihood | Mitigation Strategy                                                                                                                                                                                                                     |
| :------ | :------------- | :------------------------------------------------------------------------------------------------------------------------------------------------------------------- | :----- | :--------- | :-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **R01** | Compliance     | **GST Calculation Inaccuracies:** Errors in computing CGST/SGST vs IGST based on Place of Supply, or rounding errors leading to non-compliance.                      | High   | Medium     | Utilize `decimal.js` exclusively for math. Implement strict, mathematically sound unit tests for every tax scenario. Use `DECIMAL(15,2)` in PostgreSQL.                                                                                 |
| **R02** | Technical      | **Offline Synchronization Conflicts:** Data created offline on the Flutter mobile app conflicts with data modified on the web backend when connectivity is restored. | High   | High       | Implement deterministic conflict resolution (e.g., server-wins or last-write-wins with versioning). Limit offline creation to net-new records (invoices, expenses) rather than offline editing of complex existing records for the MVP. |
| **R03** | Security       | **Custom Authentication Vulnerabilities:** Flaws in the bespoke JWT/TOTP implementation leading to unauthorized access.                                              | High   | Low        | Follow industry standards strictly (short-lived access tokens, secure HTTP-only cookies for refresh tokens, bcrypt hashing). Conduct rigorous security reviews and penetration testing during Phase 6.                                  |
| **R04** | Operational    | **PDF Generation Bottlenecks:** Generating high-quality, professional PDF invoices on the backend consumes excessive CPU, degrading API performance.                 | Medium | Medium     | Offload PDF generation to asynchronous background jobs or worker threads. Cache static assets used in templates.                                                                                                                        |
| **R05** | Infrastructure | **Deployment Complexity:** Difficulty in managing the monorepo deployment across web, API, and mobile platforms.                                                     | Medium | Low        | Define clear Docker Compose configurations early. Ensure the mobile app (`apps/mobile`) is strictly excluded from Turborepo build steps to prevent build pipeline failures. Maintain clear deployment runbooks.                         |
| **R06** | Financial      | **Reconciliation Errors:** Discrepancies between logged expenses/invoices and the bank reconciliation module.                                                        | High   | Medium     | Enforce strict double-entry accounting principles under the hood, even if the user interface simplifies the view. Ensure database transactions (ACID) are used for all financial state changes.                                         |
| **R07** | Schedule       | **Scope Creep:** Stakeholders requesting features outside the defined 12-week MVP scope (e.g., Inventory tracking).                                                  | Medium | High       | Strictly enforce the defined MVP scope boundaries. Funnel all new feature requests into the post-MVP backlog.                                                                                                                           |
