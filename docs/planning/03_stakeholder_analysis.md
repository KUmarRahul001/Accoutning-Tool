# Stakeholder Analysis

## 1. Primary Stakeholders

| Stakeholder Group | Role / Description | Primary Needs & Expectations |
| :--- | :--- | :--- |
| **Business Owners / Management (Tinplate)** | Ultimate decision-makers and sponsors of the project. | Real-time financial visibility, accurate reporting, secure access, project delivered on time (12-week MVP), and high ROI. |
| **Accountants & Financial Officers** | Primary daily users of the web application. | Efficient data entry, automated GST calculations, robust reconciliation tools, error-free reporting, and reliable audit trails. |
| **Field Staff / Sales Representatives** | Primary users of the mobile application. | Easy-to-use mobile interface, ability to create invoices and log expenses offline, fast synchronization. |

## 2. Secondary Stakeholders

| Stakeholder Group | Role / Description | Primary Needs & Expectations |
| :--- | :--- | :--- |
| **IT Administrators** | Personnel responsible for deploying, maintaining, and supporting the system. | Easy deployment (Docker), clear documentation (Swagger, runbooks), straightforward user management, and clear logging/monitoring. |
| **External Auditors** | Third-party professionals reviewing financial compliance. | Read-only access to detailed, immutable audit logs, clear report generation, and transparent GST calculation methodologies. |

## 3. Engineering Team (Autonomous Agent Roles)

* **Software Architect:** Ensures the system meets enterprise standards, scalability, and modularity requirements.
* **Backend Engineer:** Develops secure, high-performance NestJS APIs and PostgreSQL schemas.
* **Frontend Engineer:** Crafts the responsive React/AntD web interface.
* **Mobile Engineer:** Builds the offline-first Flutter application.
* **DevOps Engineer:** Manages Docker configuration, CI/CD, and deployment strategies.
* **QA Engineer:** Ensures rigorous testing (unit, integration, GST calculation accuracy).
* **Security Engineer:** Enforces authentication, RBAC, and data protection standards.
* **Technical Writer:** Maintains Swagger documentation, READMEs, and architecture documents.
